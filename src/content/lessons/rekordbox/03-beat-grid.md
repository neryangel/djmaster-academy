# Beat Grid — ניתוח קצב, Downbeat, ותיקון ידני

## מה זה Beat Grid ולמה הוא קריטי?

Beat Grid הוא **רשת של קווים אנכיים** ש-Rekordbox מציבה על ה-Waveform של שיר. **כל קו = ביט אחד**.

### למה זה חשוב?

Beat Grid הוא **הבסיס לכל דבר:**
- **Sync** — מסנכרן BPM בין שירים לפי Grid
- **Quantize** — Cues ו-Loops נופלים בדיוק על Grid
- **Hot Cues** — קפיצות מדויקות תלויות ב-Grid
- **Loops** — לולאה שמתחילה ונגמרת על ביטים מדויקים
- **Beat Jump** — קפיצה של X ביטים בדיוק
- **Waveform Zoom** — הגדלה לפי תיבות ומשפטים

> **Grid שגוי = הכל שגוי.** Sync לא יעבוד, Loops ייצאו מקצב, ו-Cues יפלו חצי ביט לא נכון. **תמיד בדוק Grid לפני הופעה.**

---

## ניתוח אוטומטי — מה Rekordbox עושה

כשמייבאים שיר, Rekordbox:
1. **מזהה BPM** — כמה ביטים בדקה
2. **מאתר Downbeat** — ביט 1 של התיבה הראשונה
3. **מציב Grid** — קווים על כל ביט

### דיוק הניתוח לפי ז'אנר

| ז'אנר | דיוק BPM | דיוק Downbeat | צריך תיקון? |
|--------|----------|---------------|------------|
| **House / Techno** | 99% | 95% | לעיתים נדירות |
| **Trance** | 98% | 90% | לפעמים |
| **Pop / Rock** | 90% | 80% | לעיתים |
| **Hip-Hop** | 85% | 75% | כן, הרבה |
| **מוזיקה חיה** | 70% | 60% | **כמעט תמיד** |
| **מזרחי / ערבי** | 80% | 70% | הרבה פעמים |

## סימנים ש-Beat Grid שגוי

| סימן | מה קורה | הסיבה האפשרית |
|------|---------|--------------|
| **Sync לא עובד** | שירים יוצאים מקצב | BPM שגוי |
| **Loop לא ריתמי** | Loop קופץ או גומע | Grid מוזז |
| **Cue לא על ביט** | Cue נופל "באוויר" | Downbeat שגוי |
| **BPM מוזר** | 63 BPM במקום 126 | חצי/כפול BPM |
| **Grid "זז" לאורך השיר** | ביטים מתרחקים מהקווים | Dynamic BPM |

## תיקון Beat Grid — שלב אחר שלב

### שלב 1: פתח את Grid Editor

- **דאבל-קליק** על שיר → נפתח בPlayer
- לחץ על **Grid** (כפתור ליד ה-Waveform)

### שלב 2: מצא את ה-Downbeat

```
 ביט:  1   2   3   4   1   2   3   4
       ↑               ↑
       │               │
    Downbeat         Downbeat
    (Kick חזק)     (Kick חזק)

 Grid צריך להיות כאן:
       |   |   |   |   |   |   |   |
```

1. נגן את השיר
2. מצא את ה-**Kick הראשון** — זה Downbeat
3. לחץ עליו → Grid יתיישר

### שלב 3: בדוק יישור

1. נגן את השיר **קדימה**
2. ודא שכל קו נופל **בדיוק על Kick**
3. אם הGrid "זוחל" — ה-BPM שגוי:
   - **Grid בריץ קדימה** = BPM גבוה מדי → הנמך
   - **Grid מפגר מאחור** = BPM נמוך מדי → העלה

### שלב 4: כלים לתיקון

| כלי | פעולה | מתי |
|------|--------|-----|
| **Set Grid** | קובע Downbeat בנקודה הנוכחית | Downbeat שגוי |
| **Shift Grid ←→** | מזיז את כל ה-Grid ימינה/שמאלה | Grid מוזז קצת |
| **BPM ±** | שינוי BPM ב-0.01 | Grid "זוחל" |
| **½ × 2** | חצי או כפול BPM | 63 במקום 126 |
| **Reset** | מוחק Grid ומנתח מחדש | הכל השתבש |

### שלב 5: אימות סופי

- נגן את השיר **עד הסוף**
- ודא שה-Grid **לא זז** לאורך כל השיר
- אם כן — זה Dynamic BPM (ראה למטה)

## Dynamic BPM — שירים עם קצב משתנה

### מה זה?

שירים עם BPM שמשתנה במהלך השיר. נפוץ ב:
- **מוזיקה חיה** — תופים אנושיים לא מדויקים כמו מכונה
- **Transition Tracks** — שירים שעוברים מ-100 BPM ל-128 BPM
- **מזרחי/ערבי** — חלק מהשירים עם שינויי קצב

### איך להתמודד

1. **הגדר Grid ראשון** על החלק הראשון
2. **מצא את נקודת השינוי** — איפה ה-BPM משתנה
3. **הגדר Grid שני** על החלק השני
4. Rekordbox יציג **שני BPM-ים** בWaveform

---

## Quantize — שימוש נכון

### מתי ON ומתי OFF

| פעולה | Quantize | למה |
|--------|---------|-----|
| **הגדרת Hot Cues** | ✅ ON | Cue בדיוק על ביט |
| **יצירת Loops** | ✅ ON | Loop מדויק |
| **Beatmatching ידני** | ❌ OFF | לפתח דיוק אישי |
| **Scratching** | ❌ OFF | צריך חופש תנועה |
| **ניגון רגיל** | ✅ ON | Play ממשיך מביט מדויק |
